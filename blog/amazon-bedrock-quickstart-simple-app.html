How to Set Up Amazon Bedrock and Build a Simple App

A practical guide for shipping your first Bedrock-powered app in under an hour.

By spidy • Published: Mar 18, 2026 • 7 min read

Why Amazon Bedrock?

Amazon Bedrock gives you a managed way to use foundation models without managing model infrastructure. You can start with text generation quickly, then grow into guardrails, knowledge bases, and agentic workflows as your use case matures.

What We’ll Build

A small Prompt Rewriter web app:

  • User enters rough text
  • App calls Bedrock
  • Returns a polished version for email/interview/professional communication

1) Bedrock Setup (AWS Console)

  1. Open AWS Console → Amazon Bedrock.
  2. Choose a supported region (for example us-east-1).
  3. In Model access, request access to at least one text model (e.g. Claude family).
  4. Create/choose IAM user or role with Bedrock permissions.
Minimum IAM policy actions
{ "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": [ "bedrock:InvokeModel", "bedrock:InvokeModelWithResponseStream" ], "Resource": "*" } ] }

2) Local Project Setup (Node.js)

mkdir bedrock-prompt-rewriter cd bedrock-prompt-rewriter npm init -y npm install @aws-sdk/client-bedrock-runtime express dotenv
.env
AWS_REGION=us-east-1 AWS_ACCESS_KEY_ID=YOUR_KEY AWS_SECRET_ACCESS_KEY=YOUR_SECRET BEDROCK_MODEL_ID=anthropic.claude-3-haiku-20240307-v1:0 PORT=3000

3) Backend API with Bedrock

server.js
require('dotenv').config(); const express = require('express'); const { BedrockRuntimeClient, InvokeModelCommand } = require('@aws-sdk/client-bedrock-runtime'); const app = express(); app.use(express.json()); const client = new BedrockRuntimeClient({ region: process.env.AWS_REGION }); app.post('/rewrite', async (req, res) => { try { const { text } = req.body; if (!text) return res.status(400).json({ error: 'text is required' }); const prompt = `Rewrite this text in a professional and concise way:\n\n${text}`; const body = JSON.stringify({ anthropic_version: 'bedrock-2023-05-31', max_tokens: 300, messages: [{ role: 'user', content: [{ type: 'text', text: prompt }] }] }); const command = new InvokeModelCommand({ modelId: process.env.BEDROCK_MODEL_ID, contentType: 'application/json', accept: 'application/json', body }); const response = await client.send(command); const decoded = JSON.parse(new TextDecoder().decode(response.body)); const output = decoded?.content?.[0]?.text || 'No response'; res.json({ output }); } catch (error) { console.error(error); res.status(500).json({ error: 'Bedrock call failed' }); } }); app.listen(process.env.PORT || 3000, () => { console.log('Server running on port', process.env.PORT || 3000); });

4) Tiny Frontend (Optional)

index.html
<textarea id="input" rows="7" placeholder="Paste rough text..."></textarea> <button onclick="rewrite()">Rewrite</button> <pre id="out"></pre> <script> async function rewrite() { const text = document.getElementById('input').value; const res = await fetch('/rewrite', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ text }) }); const data = await res.json(); document.getElementById('out').textContent = data.output || data.error; } </script>

5) Run It

node server.js # open http://localhost:3000

Now type rough content and test the rewrite API. You’ve built a real Bedrock app.

Production Tips

  • Don’t expose keys in frontend — keep Bedrock calls server-side.
  • Add request limits to protect from abuse and cost spikes.
  • Log model latency + token usage to monitor cost/perf.
  • Add guardrails for harmful/off-topic outputs before showing users.
  • Use model fallbacks for reliability during regional/model issues.

Common Errors and Fixes

  • AccessDeniedException: model access not approved or IAM policy missing.
  • ValidationException: body format doesn’t match model schema.
  • Region mismatch: model access enabled in one region, app calling another.

Final Thought

Start with one focused workflow like rewriting, summarization, or classification. Once stable, expand into knowledge bases, guardrails, and eventually agentic capabilities. Bedrock makes this progression clean and enterprise-friendly.
