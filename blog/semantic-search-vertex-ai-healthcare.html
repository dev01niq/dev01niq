Semantic Search for Healthcare Patient Records with Vertex AI (Node.js + NestJS)

How to build an intelligent search layer that understands medical meaning—not just keywords—while staying practical for production healthcare systems.

By spidy • Published: Mar 4, 2026 • 8 min read

Why Semantic Search Matters in Healthcare

In healthcare projects, traditional keyword search often misses critical context. A doctor may search for "heart attack history", while records contain terms like "myocardial infarction". A strict keyword engine can fail here. Semantic search solves this by comparing meaning instead of exact wording.

With semantic search, clinicians and operations teams can find patient records faster, reduce manual lookup effort, and improve decision quality during time-sensitive workflows.

Simple Architecture (Production Friendly)

  1. Store patient record summaries (de-identified where needed).
  2. Generate embeddings using Vertex AI Embeddings.
  3. Store vectors in a vector store (Vertex AI Vector Search, pgvector, etc.).
  4. At query time, embed user query and run nearest-neighbor search.
  5. Return top matches + metadata (facility, date range, diagnosis tags).

Data Modeling Tip for Patient Records

Don’t embed raw EHR dumps. Create a normalized text document per searchable unit, for example:

{ "recordId": "rec_10293", "patientId": "pat_991", "text": "72-year-old female, history of myocardial infarction, hypertension, type 2 diabetes, recent complaint: chest discomfort after exertion." }

This gives cleaner embeddings and more reliable retrieval quality.

NestJS Service Example: Generate Embeddings (Vertex AI)

Below is a minimal service pattern. In production, add retries, circuit breakers, and structured logging. Also, prefer selecting the currently recommended Vertex text embedding model for your region/environment, since model versions evolve over time.

src/search/embedding.service.ts
import { Injectable } from '@nestjs/common'; import { PredictionServiceClient } from '@google-cloud/aiplatform'; @Injectable() export class EmbeddingService { private client = new PredictionServiceClient(); private projectId = process.env.GCP_PROJECT_ID!; private location = process.env.GCP_REGION || 'us-central1'; private model = 'textembedding-gecko@003'; async embedText(text: string): Promise { const endpoint = `projects/${this.projectId}/locations/${this.location}/publishers/google/models/${this.model}`; const [response] = await this.client.predict({ endpoint, instances: [{ content: text } as any], parameters: {} as any, }); const values = response.predictions?.[0] as any; return values?.embeddings?.values ?? []; } }

Healthcare Guardrails You Should Not Skip

  • PHI handling: Minimize PII in embedding text where possible.
  • Access filters: Always enforce tenant/facility-level filters in vector queries.
  • Auditability: Log who queried what, and when.
  • Prompt safety: If used with RAG, prevent leaking restricted context in generated responses.
  • Model fallback: Add graceful degradation if embedding service is unavailable.

Practical KPI Targets

For healthcare use-cases, measure these from day one:

  • Search relevance (Top-3 hit rate)
  • Latency P95 for user query to first result
  • Zero unauthorized retrieval incidents
  • Reduction in manual chart lookup time

Common Pitfalls to Avoid

  • Embedding raw PHI blindly: redact or minimize sensitive direct identifiers before indexing.
  • No tenant/facility filters: always apply access constraints during vector retrieval.
  • No audit trail: log semantic search usage for security and compliance reviews.
  • Ignoring relevance QA: evaluate with real clinical phrasing, not only synthetic queries.

Compliance Note

This article is for engineering education and architecture guidance. Real deployments handling patient data must align with applicable regulations (such as HIPAA/PHIPA/GDPR), internal security controls, and legal/compliance review requirements.

Final Thoughts

Semantic search with Vertex AI can be a high-impact capability for healthcare products when built with strong access controls and clean data preparation. Start simple: one embedding pipeline, one searchable patient-summary index, and clear governance. Then iterate with relevance feedback from clinicians.

That approach gives teams a practical path from pilot to production without overengineering from day one.
